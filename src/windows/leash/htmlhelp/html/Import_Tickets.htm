Import Tickets

You can import Windows domain tickets that you have already obtained through a Windows Logon session. Imported tickets can be fully used by applications that require the MIT Kerberos interface. In most installations, MIT Kerberos will automatically import these tickets if possible.

Turn Automatic Import on or off

In most installations, MIT Kerberos will automatically import tickets if possible. Go to the Options tab and click the Automatic Import Tickets checkbox in the Ticket Options panel to turn the feature on or off. Using the Options Panel

In some cases MIT Kerberos tries to automatically import tickets but is prevented from doing so by the Windows User Access Control (UAC) feature. If this happens the tickets are still displayed in the main window, but have the Import Status of protected. You can turn off UAC on your computer to allow the tickets to be imported.

Read ticket Import Status

If you have tickets obtained when you logged on to a Windows domain with a Windows Logon session, the main window of MIT Kerberos shows the Import Status for all of your tickets.

  • imported tickets have already been imported into MIT Kerberos.
  • importable tickets have not been imported into MIT Kerberos but can be. To import them, either turn on the Automatic Import Tickets option in the Options tab or click the Import Tickets button in the Home tab.
  • protected tickets cannot currently be imported into MIT Kerberos because the Windows User Access Control (UAC) is preventing this action. You can turn off UAC on your computer to allow the tickets to be imported.

Use the Import Ticket button

If the Automatic Import option is turned off, you can still import tickets.

  1. Go to the Home tab.
  2. Click the Import Tickets button.
  3. Click Okay to confirm that you want to import your tickets and destroy any that are already in MIT Kerberos.
