Import Tickets

You can import Windows domain tickets that you have already obtained through a Windows Logon session. Imported tickets can be fully used by applications that require the Kerberos for Windows interface. In most installations, Kerberos for Windows will automatically import these tickets if possible.

Turn Automatic Import on or off

In most installations, Kerberos for Windows will automatically import tickets if possible. Click the Automatic Import checkbox in the Options panel to turn the feature on or off. Using the Options Panel

In some cases Kerberos for Windows tries to automatically import tickets but is prevented from doing so by the Windows User Access Control (UAC) feature. If this happens the tickets are still displayed in the main window, but have the Import Status of protected. You can turn off UAC on your computer to allow the tickets to be imported.

Read ticket Import Status

If you have tickets obtained when you logged on to a Windows domain with a Windows Logon session, the main window of Kerberos for Windows shows the Import Status for all of your tickets. You can hide or show this column with the View panel checkboxes. Using the View Panel

  • imported tickets have already been imported into Kerberos for Windows.
  • importable tickets have not been imported into Kerberos for Windows but can be. To import them, either turn on the Automatic Import option or click Import Tickets in the More panel.
  • protected tickets cannot currently be imported into Kerberos for Windows because the Windows User Access Control (UAC) is preventing this action. You can turn off UAC on your computer to allow the tickets to be imported.

Use the Import Ticket button

If the Automatic Import option is turned off, you can still import tickets.

  1. Find the More panel at the right edge of the ribbon menu.
  2. Click the More button to see your choices and select Import Tickets.
  3. Click Okay to confirm that you want to import your tickets and destroy any that are already in Kerberos for Windows.
